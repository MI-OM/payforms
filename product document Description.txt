🧱 PAYFORM – SYSTEM ARCHITECTURE (FINAL)
🎯 1. ARCHITECTURE PRINCIPLES
Multi-tenant SaaS (organization-based isolation)
No fund custody (org uses Paystack)
Modular Monolith (fast MVP, scalable later)
API-first backend
Event-driven internally (lightweight)
Separation of concerns across modules

🔁 4. END-TO-END DATA FLOW
A. FORM CREATION FLOW
Admin → Create Form
      → Add Fields
      → Assign Targets (Group/Contact)
      → Publish

Stored in:
- forms
- form_fields
- form_targets
B. PAYMENT FLOW (CRITICAL PATH)
User → Fill Form → Submit

Backend:
→ Create submission
→ Create payment record
→ Initialize Paystack payment

User → Pay via Paystack

Paystack:
→ Redirect to callback
→ Send webhook

Backend:
→ Verify payment
→ Update payment status
→ Log event
→ Send notification
C. CONTACT MANAGEMENT FLOW
Admin → Import Contacts (CSV)
      → Map Fields
      → Validate
      → Save

Contacts:
→ Assigned to groups
→ Can login (optional)
→ Can view payment history
D. TARGETING FLOW
Form → Assigned to:
   - Group(s)
   - Subgroup(s)
   - Contact(s)

Backend resolves:
→ All eligible contacts
→ Used for:
   - Access control
   - Reporting
   - Notifications
